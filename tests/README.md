# Автоматические тесты Dance Studio

Этот набор тестов включает API тесты, unit тесты и интеграционные тесты для приложения Dance Studio.

## Структура тестов

```
tests/
├── api/                    # API тесты
│   ├── auth.test.js       # Тесты авторизации
│   ├── admin.test.js      # Тесты административных функций
│   └── schedule.test.js   # Тесты расписания
├── unit/                   # Unit тесты
│   ├── auth.utils.test.js # Тесты утилит авторизации
│   └── schedule.utils.test.js # Тесты утилит расписания
├── integration/            # Интеграционные тесты
│   └── user-flow.test.js  # Тесты пользовательских сценариев
├── package.json           # Зависимости тестов
├── jest.config.js        # Конфигурация Jest
└── setup.js              # Глобальная настройка тестов
```

## Установка и запуск

### 1. Установка зависимостей

```bash
cd tests
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта или настройте переменные:

```bash
API_URL=http://localhost:5000
FRONTEND_URL=http://localhost:3000
```

### 3. Запуск серверов

Убедитесь, что backend и frontend серверы запущены:

```bash
# Терминал 1
cd backend
npm start

# Терминал 2  
cd frontend
npm start
```

Или используйте Docker:
```bash
docker-compose up
```

### 4. Запуск тестов

#### Все тесты:
```bash
npm test
```

#### Только API тесты:
```bash
npm run test:api
```

#### Только unit тесты:
```bash
npm run test:unit
```

#### Только интеграционные тесты:
```bash
npm run test:integration
```

#### Тесты с покрытием кода:
```bash
npm run test:coverage
```

#### Тесты в режиме watch:
```bash
npm run test:watch
```

## Устранение неполадок

### Тесты не запускаются
1. Проверьте, что установлены все зависимости: `npm install`
2. Убедитесь, что серверы запущены на правильных портах
3. Проверьте переменные окружения

### API тесты failing
1. Проверьте доступность backend сервера: `curl http://localhost:5000/api/health`
2. Убедитесь, что база данных работает
3. Проверьте наличие тестовых пользователей в БД

### Unit тесты failing
1. Проверьте корректность импортов
2. Убедитесь, что все моки правильно настроены

### Интеграционные тесты failing
1. Проверьте, что все сервисы работают
2. Убедитесь в наличии тестовых данных
3. Проверьте права доступа тестовых пользователей

## Добавление новых тестов

1. Создайте новый файл в соответствующей директории (api/, unit/, integration/)
2. Используйте существующие тесты как шаблон
3. Следуйте соглашениям об именовании: `*.test.js`
4. Используйте глобальные хелперы из `setup.js`

### Пример нового API теста:

```javascript
const { describe, test, expect } = require('@jest/globals');

describe('Новый функционал', () => {
  test('Тест новой функции', async () => {
    const response = await global.apiClient.get('/api/new-endpoint');
    
    expect(response.status).toBe(200);
    expect(response.data).toHaveProperty('data');
  });
});
```

## CI/CD интеграция

Для интеграции с GitHub Actions или другими CI системами:

```yaml
- name: Run tests
  run: |
    cd tests
    npm install
    npm run test:coverage
```

Тесты автоматически проверяют работоспособность API и бизнес-логики приложения.
