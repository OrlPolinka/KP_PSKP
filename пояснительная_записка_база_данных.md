# Пояснительная записка к структуре базы данных проекта "Dance Studio"

## 2. Структура базы данных

### 2.1 Общее описание

База данных проекта "Dance Studio" разработана для управления танцевальной студией, включая управление пользователями, расписанием занятий, записями клиентов, абонементами, платежами и системой чата. База данных построена на PostgreSQL и использует ORM Prisma для взаимодействия с приложением.

### 2.4 Индексы базы данных

Для оптимизации производительности запросов в базе данных созданы следующие индексы:

- **MESSAGES**: составной индекс по полям (senderid, receiverid) для быстрого поиска переписок
- **MESSAGES**: индекс по полю createdat для сортировки сообщений
- **NOTIFICATIONS**: составной индекс по полям (userid, isread) для фильтрации непрочитанных уведомлений
- **NOTIFICATIONS**: индекс по полю createdat для сортировки уведомлений
- **BOOKINGS**: индекс по полю scheduleid для быстрого поиска записей на занятие
- **BOOKINGS**: индекс по полю clientid для поиска записей клиента

### 2.5 Ограничения целостности данных

В базе данных определены следующие ограничения для обеспечения целостности данных:

- **Внешние ключи**: все связи между таблицами реализованы через внешние ключи с каскадным удалением
- **Уникальные индек**: email в таблице USERS, name в таблице DANCESTYLES
- **NOT NULL ограничения**: обязательные поля для заполнения
- **CHECK ограничения**: валидация значений полей (статусы, роли и т.д.)

### 2.6 Безопасность данных

- **Пароли**: хранятся в хешированном виде с использованием bcrypt
- **Токены**: JWT токены для аутентификации с ограниченным сроком действия
- **Платежные данные**: не хранятся в системе, используются токены платежных провайдеров
- **QR-коды**: генерируются с уникальными идентификаторами и сроком действия

## 3. Используемые технологии

### 3.1 Система управления базами данных

- **PostgreSQL 16-alpine**: основная СУБД для хранения данных
- **Prisma ORM**: для взаимодействия с базой данных и управления миграциями
- **UUID**: использование универсальных идентификаторов для обеспечения безопасности

### 3.2 Backend технологии

- **Node.js 22.x**: среда выполнения JavaScript на сервере
- **Express.js 5.2.1**: фреймворк для создания REST API
- **Socket.IO 4.8.3**: для реализации реального времени в чате
- **JWT**: для аутентификации и авторизации пользователей
- **bcrypt**: для хеширования паролей
- **multer**: для загрузки файлов изображений

### 3.3 Frontend технологии

- **React 19.2.4**: библиотека для создания пользовательского интерфейса
- **React Router 7.14.0**: для навигации между страницами
- **Axios**: для HTTP запросов к backend
- **Socket.IO Client**: для реального времени в чате
- **Bootstrap 5.3.3**: CSS фреймворк для стилизации
- **Lucide React**: библиотека иконок

### 3.4 Инфраструктура и развертывание

- **Docker**: контейнеризация приложения
- **Docker Compose**: оркестрация контейнеров
- **NGINX**: reverse proxy для распределения запросов
- **Ubuntu Server 24.04 LTS**: операционная система сервера

### 3.5 Платежные системы

- **Stripe**: интеграция для приема платежей
- **YooKassa**: альтернативная платежная система
- **Webhooks**: для обработки статусов платежей

### 3.6 Дополнительные сервисы

- **QR-код генерация**: библиотека qrcode для создания QR-кодов
- **Google OAuth**: аутентификация через Google аккаунт
- **Email уведомления**: система отправки уведомлений пользователям
