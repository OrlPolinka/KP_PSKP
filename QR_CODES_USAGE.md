# Система QR-кодов для посещения занятий

## Обзор системы

Система QR-кодов позволяет клиентам генерировать уникальные QR-коды для своих занятий, а тренерам - сканировать их для отметки посещаемости.

## Для клиентов

### 1. Генерация QR-кодов

Клиенты могут получить QR-коды двумя способами:

#### Через страницу "QR-коды" в меню:
1. Войдите в систему как клиент
2. В левом меню выберите "QR-коды"
3. На вкладке "На сегодня" увидите все сегодняшние занятия
4. Нажмите "Показать QR-код" для нужного занятия
5. Покажите QR-код тренеру при приходе

#### Через конкретное бронирование:
1. На странице "QR-коды" перейдите на вкладку "По ID бронирования"
2. Введите ID бронирования
3. Нажмите "Сгенерировать QR-код"

### 2. Особенности QR-кодов

- QR-коды генерируются автоматически при создании бронирования
- Каждый QR-код уникален и привязан к конкретному занятию
- QR-код можно использовать только в день занятия
- После отметки посещения QR-код становится неактивным

## Для тренеров

### 1. Сканирование QR-кодов

1. Войдите в систему как тренер
2. В левом меню выберите "Сканер QR"
3. Нажмите "Начать сканирование"
4. Разрешите доступ к камере браузера
5. Наведите камеру на QR-код клиента
6. Дождитесь автоматического распознавания

### 2. Отметка посещения

После успешного сканирования:
1. Проверьте информацию о клиенте и занятии
2. Нажмите "Отметить посещение" если клиент пришел
3. Нажмите "Не пришел" если клиент отсутствует
4. Система автоматически обновит статус посещения

### 3. Правила безопасности

- Тренер может отмечать посещения только своих занятий
- Сканирование доступно только в день занятия
- Нельзя отметить посещение ранее чем за 15 минут до начала
- Каждое сканирование логируется в системе

## API эндпоинты

### Для клиентов

- `GET /bookings/today/qrcodes` - Получить QR-коды на сегодня
- `GET /bookings/:bookingId/qrcode` - Получить QR-код конкретного бронирования

### Для тренеров

- `POST /bookings/verify-qrcode` - Проверить QR-код
- `POST /bookings/mark-attendance-by-qr` - Отметить посещение через QR-код

## Структура данных QR-кода

```json
{
  "bookingId": "uuid-бронирования",
  "qrCode": "уникальный-код",
  "scheduleId": "uuid-занятия",
  "clientId": "uuid-клиента"
}
```

## Обработка ошибок

### Возможные ошибки и их решения:

1. **"QR-код не найден"**
   - QR-код недействителен или отсутствует в системе
   - Решение: Попросите клиента сгенерировать новый QR-код

2. **"Это не ваше занятие"**
   - Тренер пытается отметить чужое занятие
   - Решение: Убедитесь, что это ваше занятие в расписании

3. **"Занятие не сегодня"**
   - Попытка отметить посещение не в день занятия
   - Решение: Отмечайте посещения только в день занятия

4. **"Нельзя отметить посещение до начала занятия"**
   - Попытка отметить посещение ранее чем за 15 минут до начала
   - Решение: Подождите до разрешенного времени

5. **"Посещение уже отмечено"**
   - Повторная попытка отметить то же посещение
   - Решение: Посещение уже отмечено в системе

## Технические требования

### Для корректной работы сканера:
- Современный браузер с поддержкой WebRTC
- Разрешение на использование камеры
- Стабильное интернет-соединение
- HTTPS (для работы камеры в некоторых браузерах)

### Рекомендуемые браузеры:
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## Безопасность

- Все QR-коды содержат криптографически случайные данные
- Время жизни QR-кода ограничено днем занятия
- Каждое действие логируется с указанием тренера и времени
- Дополнительная проверка прав доступа тренера

## Поддержка

При возникновении проблем с QR-кодами:
1. Проверьте интернет-соединение
2. Убедитесь, что используете поддерживаемый браузер
3. Попробуйте обновить страницу
4. Обратитесь к администратору системы
